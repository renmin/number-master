# "不要碰比我大的数字" 美术设计方案

## 1. 整体风格定义

### 1.1 美术风格
**3D 低多边形（Low Poly）卡通风格**

- 色彩明快饱和、视觉干净清爽
- 几何化造型，简洁不复杂
- 风格对标：跑酷类小游戏主流审美（如 Number Master、Tall Man Run）
- 目标感受：轻松、解压、有趣、可爱

### 1.2 风格关键词
`明亮` `圆润` `卡通` `扁平化3D` `糖果色` `简洁` `动感`

## 2. 色彩体系

### 2.1 主色板

| 用途 | 颜色 | Hex值 | 说明 |
|------|------|-------|------|
| 可吞噬数字（安全色） | 天蓝色 | #4FC3F7 | 友好、安全感 |
| 危险数字（警告色） | 珊瑚红 | #FF6B6B | 警告、危险感 |
| 玩家数字 | 明黄色 | #FFD93D | 主角、醒目 |
| 跑道主色 | 浅灰白 | #F5F5F5 | 干净的跑道底色 |
| 跑道边线 | 薄荷绿 | #6BCB77 | 清新活力 |
| UI主色 | 深紫蓝 | #4A3AFF | 科技感、现代 |
| UI辅助色 | 橙色 | #FF8C42 | 按钮高亮、提示 |
| 背景渐变 | 浅蓝→白 | #E3F2FD→#FFF | 天空感、开阔 |

### 2.2 颜色应用规则
- 蓝色系 = 安全、可吞噬 → 让玩家建立条件反射
- 红色系 = 危险、不可碰 → 强烈视觉警告
- 黄色系 = 玩家自身 → 时刻能在画面中找到自己
- 数字越大颜色越深（深红=极度危险）
- 数字越小颜色越浅（浅蓝=轻松吞噬）

## 3. 角色设计

### 3.1 玩家角色

**基本形态：圆润的数字球体**

```
设计描述：
- 3D球形/圆柱体，表面显示当前数字
- 数字字体：粗圆无衬线体（如 Rounded Bold）
- 表情系统：有简单的眼睛和嘴巴表情
  - 正常：微笑 :)
  - 吞噬时：张嘴 :D
  - 接近危险：紧张 :O
  - 失败时：哭泣 :(
- 尺寸随数字增大而略微变大（有上限）
- 移动时有弹跳感和拖尾粒子效果
```

## 6. 动效设计

### 6.1 核心动效

| 动作 | 动效描述 | 时长 |
|------|---------|------|
| 数字吞噬 | 小数字被吸入→消失→玩家数字放大弹跳→回落 | 0.3s |
| 碰撞失败 | 红色闪烁+震屏+碎裂粒子+数字归零动画 | 0.5s |
| 数字增长 | 数字跳动变化（老虎机滚动效果） | 0.2s |
| 墙壁破碎 | 碎片向四面八方飞散+冲击波+光效 | 0.4s |
| Combo | 连击文字弹出放大→淡出，金色粒子 | 0.5s |
| 道具激活 | 环形光波扩散+角色短暂发光 | 0.3s |

### 6.2 转场动效
- 关卡切换：画面向上滑出 → 新关卡从下方滑入
- 菜单切换：卡片式左右滑动
- 弹窗出现：从中心放大+弹性回弹（overshoot）

### 6.3 粒子效果
| 场景 | 粒子类型 |
|------|---------|
| 吞噬数字 | 蓝色星形碎片飞散 |
| 碰撞失败 | 红色心碎粒子+烟雾 |
| 破墙 | 砖块碎片+灰尘+光芒 |
| 冲刺 | 尾部速度线+风粒子 |
| 道具拾取 | 金色光圈上升消散 |

## 10. 设计原则总结

1. **一眼看懂** — 蓝色=安全，红色=危险，不需要文字说明
2. **正反馈强** — 吞噬时有明显的视觉+音效+手感反馈
3. **节奏感好** — 数字增长的爽感要做足，失败的挫败感要快速恢复
4. **性能优先** — 视觉效果服从于流畅体验，低端机也要保证60fps
5. **易于迭代** — 模块化设计，便于添加新主题、皮肤、关卡
