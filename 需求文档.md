## 2. 核心玩法

### 2.1 基本规则
1. 玩家控制一个带有数字的角色/方块在跑道上自动向前移动
2. 跑道上分布着各种带数字的目标：
   - **蓝色数字**：比玩家当前数字小 → 可以吞噬，玩家数字增加
   - **红色数字**：比玩家当前数字大 → 碰到即失败，重新开始
3. 玩家通过左右滑动来选择路线，躲避红色大数字，吞噬蓝色小数字
4. 目标是让数字尽可能变大，到达终点线

### 2.2 操作方式
| 平台 | 操作 |
|------|------|
| 手机 | 手指左右滑动控制角色移动方向 |
| PC | 鼠标左右拖动 / 键盘A/D或左右方向键 |

### 2.3 胜利条件
- 安全到达关卡终点线
- 终点后有多面墙壁，用累积的数字逐一击破，摧毁越多墙壁分数越高

### 2.4 失败条件
- 碰撞到比自己大的红色数字
- 碰到障碍物（电锯等）

## 3. 游戏系统

### 3.1 关卡系统
| 要素 | 描述 |
|------|------|
| 关卡总数 | 初版50关，后续持续更新 |
| 难度递进 | 每隔10关提升难度等级 |
| 难度体现 | 数字范围扩大、障碍物增多、路线变窄、速度加快 |
| 特殊关卡 | 每10关设置Boss关（巨型数字墙） |

### 3.2 数字系统
| 难度等级 | 玩家初始值 | 场景数字范围 | 终点墙壁数 |
|-----------|-----------|-------------|-----------|
| 1-10关 | 1-5 | 1-50 | 3-5面 |
| 11-20关 | 1-10 | 1-200 | 5-8面 |
| 21-30关 | 1-20 | 1-500 | 8-12面 |
| 31-40关 | 1-50 | 1-1000 | 12-15面 |
| 41-50关 | 1-100 | 1-2000 | 15-20面 |

### 3.3 障碍系统
| 障碍类型 | 出现关卡 | 效果 |
|----------|---------|------|
| 电锯 | 5关起 | 碰到即失败 |
| 沟渠 | 10关起 | 需要跳跃通过 |
| 桥梁 | 15关起 | 窄通道，限制闪避空间 |
| 加速带 | 20关起 | 加速通过，增加难度 |
| 分叉路 | 25关起 | 选择不同路线，奖励不同 |

### 3.4 道具系统
| 道具 | 效果 | 持续时间 |
|------|------|---------|
| 护盾 | 免疫一次碰撞 | 一次性 |
| 磁铁 | 自动吸附附近小数字 | 5秒 |
| 2倍增长 | 吞噬数字时翻倍增长 | 5秒 |
| 减速 | 降低移动速度便于操作 | 3秒 |

### 3.5 评分系统
| 评分维度 | 计算方式 |
|----------|---------|
| 基础分 | 到达终点时的数字值 x 10 |
| 墙壁奖励 | 每摧毁一面墙 +100分 |
| 连续吞噬 | 连续吞噬Combo加成（x1.5, x2, x3） |
| 时间奖励 | 快速通关额外加分 |
| 星级评定 | 1-3星，基于总分阈值 |

### 3.6 成长系统
| 系统 | 描述 |
|------|------|
| 皮肤收集 | 解锁不同数字外观/特效 |
| 成就系统 | 完成挑战获得成就徽章 |
| 排行榜 | 全球/好友排行 |
| 每日挑战 | 每日特殊关卡 |

## 5. 技术需求

| 指标 | 要求 |
|------|------|
| 帧率 | 60fps（中高端）/ 30fps（低端） |

| 内存占用 | <200MB |
| 兼容设备 | iOS 12+ / Android 7.0+ |

### 5.2 技术选型建议
| 组件 | 推荐方案 |
|------|---------|
| 游戏引擎 | Unity 2022+ / Cocos Creator 3.x |
| 渲染 | 3D低多边形风格 / 2.5D视角 |
| 物理 | 简单AABB碰撞检测 |
| 数据存储 | 本地PlayerPrefs + 云端存档 |
| 广告SDK | AdMob / 穿山甲 / Unity Ads |
| 分析 | Firebase Analytics / 友盟 |

## 7. 风险评估

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 玩法同质化 | 用户留存低 | 增加创新机制（如数字合成、技能） |
| 难度曲线过陡 | 新用户流失 | 动态难度调整，新手引导 |
| 广告过多 | 用户体验差 | 控制广告频率，提供去广告选项 |
| 性能问题 | 低端机卡顿 | 多级画质设置，资源优化 |
